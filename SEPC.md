# 介绍

一个美术人员使用的 Git 桌面端工具，名称叫 Artistic Git。

- 它基于 Tauri + React + shadcn ui + lucide icon + Vite + Vitest，全部技术栈均使用最新稳定版本。
- 支持自动更新，运行时每小时检查更新，并有主动检查更新按钮。
- 支持 Github Actions 编译构建，并支持手动触发和推送提交自动触发。
- 版本号自动递增，自动上传二进制文件到 Github Releases，自动收集提交信息作为更新日志。
- 需要编译所有平台的产物。
- 对于核心逻辑需要编写严格的单元测试和端到端测试（不关心测试时长，用真实临时项目测试，而不是 fake/mock 命令），该工具涉及到资产丢失问题，需要首要保证安全性和稳定性。
- 当代码发生非预期错误时禁止静默处理。
- 记录详细错误日志，发生错误时弹出专门的错误详情弹窗，并支持复制错误信息和打开日志文件所在目录方便发送给开发人员。
- 如果错误会影响到工具之后的运行（例如状态异常），必须弹出专门的奔溃详情弹窗，提供按钮让用户重启工具。
- 界面响应式设计，支持任意窗口尺寸下最佳显示，默认尺寸 1280x720。

# 设计总则

- 设计上应简单人性化，美术人员对 Git 技术并不了解，在使用过程中应尽量避免存在对 Git 概念和操作理解的要求，将主要工作流设计为可视化操作，在工具内部自动执行 Git 命令。
- 界面应简洁精美，有苹果和 Linear 的设计感，多使用图标以增加亲近感和友好性。
- 合理使用符合直觉、苹果设计风格的动画效果。
- 超出长度的文本使用简略显示，如："/sdfsdf/sdfsd/fsdf..."， 鼠标悬停弹出 tooltip 显示完整路径。
- 保持界面元素自身文案的简练性，需要更多说明则使用悬停 tooltip 展示。
- 支持同时打开多个窗口管理多个项目。
- 支持中英文双语切换，自动根据系统语言切换，支持手动切换语言。
- 支持深色和浅色主题切换，自动根据系统主题切换，支持手动切换主题。
- 所有破坏性操作均需要弹出二次确认提示框。

# 独特设计

- 工具内嵌 Git，所有 Git 操作均使用内嵌 Git 执行。
- 远程仓库地址：工具会默认仓库只有一个远程仓库地址，名称为 `origin`，不考虑存在多个远程仓库的情况。
- 内部会实时 Fetch 远程分支。
- 分支：显示分支时将本地分支和远程分支合并显示，远程分支名称会去掉前缀 `origin/`，因为并不考虑存在多个远程仓库的情况。
- 本地更改：不区分暂存区和工作区，修改、新增、删除，所有未提交的更改都统一称为本地更改。
- 创建分支：创建时，需用户输入分支名称和基于的分支，默认基于当前所在分支；并且复选框：是否远程分支，默认勾选；是否立即切换到该分支，默认勾选。
- 删除分支：删除时，需用户选择是否删除远程分支，默认不勾选。
- 同步：默认情况下，同步操作会根据设置中是否开启自动跟踪来决定具体流程。
    - 自动跟踪：可以设置 A 分支追踪 B 分支，设置了自动追踪的 A 分支在执行同步操作时会使用不同的流程。
    - 开启自动跟踪时：
        - 如果 A 分支有本地更改，则自动储藏。
        - 拉取 A 和 B 分支。
        - 自动 ff only 合并 B 分支到 A 分支，如果失败则提示，然后进入恢复储藏流程。
        - 推送 A 分支到远程仓库。
        - 进入恢复储藏流程。
    - 关闭自动跟踪时：
        - 如果 A 分支有本地更改，则自动储藏。
        - 拉取 A 分支。
        - 进入恢复储藏流程。
- 储藏：如果是软件内部自动储藏，则名称以 “Auto Stash:” 开头，并说明理由，并且整个流程成功后删除储藏，否则保留储藏。
- 恢复储藏：
    - 如果能够成功恢复储藏，则无需任何额外操作。
    - 如果储藏的更改和当前分支有冲突，则弹出冲突解决界面。
- 切换分支：
    - 如果本地有修改，切换分支时弹出确认框，让用户选择如何处理：
        1. 自动储藏并恢复本地更改。（默认）
        2. 丢弃本地更改。
- 提交更改：
    - 自动先同步，直到成功再进行下一步骤，否则中断。
    - 提交的二次确认框中有复选框，是否立即推送到远程仓库，默认勾选。
    - 执行提交，如果因为未同步等可通过重试的错误失败，则再次自动同步。
- 撤回提交：
    - 撤回的二次确认框中有复选框，是否立即推送到远程仓库，默认勾选。
    - 仅能够对当前分支的提交进行撤回，否则提示要切换到对应分支。
    - 撤回前会自动同步，直到成功再进行下一步骤，否则中断。
    - 撤回前自动储藏本地更改。
    - 执行撤回，如果因为未同步等可通过重试的错误失败，则再次自动同步。
- 审查模式：
    - 在主界面点击 “审查模式” 按钮，进入审查模式，此时覆盖层阻止所有操作，中间有一个 “退出审查模式” 按钮。
    - 审查模式即会先将本地修改自动储藏，然后同步以保持本地和远程一致状态。
    - 退出审查模式时会自动恢复储藏的本地修改。
- 以上所有自动多步骤融合操作首先要确保操作有备份，如果中途任何一个步骤是无法通过重试解决的失败，则提示，并恢复到操作前的状态，否则自动重试。

# 起始界面

- 打开该工具判断是否首次进入，是则打开向导界面，否则打开起始界面。
- 左侧是两个带图标的文字按钮，克隆项目、打开项目。
- 右侧是之前打开过的项目列表，每个列表项左边是图标，然后经跟着图标的是左对齐的两行文本：目录名称和灰色字体的详细路径。点击可以打开项目。
鼠标悬停在列表项上，会出现一排右对齐的按钮（仅图标）组，分别是：删除。
列表的最底部有清空历史记录的文字链接形式的按钮。
- 界面的右下角是右对齐的按钮（仅图标）组，从左到右是：向导、设置。

# 向导界面

- 作者信息
    - 要求用户输入 Git 使用的用户名和邮箱地址，这会存储起来，工具内部的操作均使用这些信息。
- SSH Key
    - 检测用户是否有 Git 能使用的 SSH Key，如果没有则显示生成 SSH Key 按钮，并附上 SSH Key 作用说明。
    点击按钮会弹出窗口让用户输入信息以生成 SSH Key（尽量提供合理、常见的默认值让用户少输入）。
    - 生成后、或已有 SSH Key 的情况下，显示复制 SSH Key 的按钮，并附上将 Key 添加到 Git 平台的小字说明文本，点击后复制到剪贴板。

# 主界面

- 选择项目后进入主界面。
- 主界面分为左侧栏和主面板区域。
- 左侧栏内容从上到下排列：
    - 项目信息区域，从左到右是图标；文本区域，文本区域是左对齐的两行文本：目录名称和灰色字体的详细路径；右对齐的一个同步按钮（仅图标），如果存在可同步的内容则会变成橙色，否则是正常主色。
    - 分支信息区域，列表的形式展示分支，每个列表项占一行，从左到右是：当前所在分支标记（点）、图标、分支名称，然后是右对齐的待同步提交数 Badge（如果没有待同步提交则不显示）。
        - 图标：本地分支显示为分支图标，远程分支显示为云分支图标。
        - 区域标题左对齐，同行右对齐是按钮（仅图标）组，分别是：新建分支。
        - 鼠标悬停在列表项上，会出现一排右对齐的按钮（仅图标）组，分别是：同步，切换，删除。
        - 点击分支会将主面板切换到历史选项卡，并聚焦到该分支的最新提交。
        - 右键列表项会弹出菜单，菜单内容为：同步，切换，作为基准创建新分支，删除。
    - 储藏信息区域，列表的形式展示储藏记录，每个列表项占一行，从左到右是：图标、储藏名称，然后是右对齐的储藏时间。
        - 区域标题左对齐，同行右对齐是按钮（仅图标）组，分别是：创建储藏。
        - 鼠标悬停在列表项上，会出现一排右对齐的按钮（仅图标）组，分别是：应用、删除、详情。
    - 分支信息和储藏信息区域均可以折叠，并有搜索框可以进行模糊搜索。
        - 未搜索到时会显示 “未搜索到相关内容” 的提示。
    - 审查模式按钮区域，有一个撑满该区域的青色渐变的 “进入审查模式” 按钮，点击后进入审查模式。
    - 更多操作按钮区域，从左到右排列的按钮（仅图标）组：设置。
    - 项目信息区域、审查模式按钮区域、更多操作按钮区域是固定高度的，分支信息和储藏信息区域本身的高度是自动撑满可用左侧栏高度的，并且有 handle 可以调节区域占用的尺寸，列表内容过多时会出现滚动条。
    调节的比例会被记录，下次打开项目时会恢复上次的比例。
- 主面板区域最上方有两个选项卡进行切换，默认显示 "历史" 选项卡，另一个是 "本地更改"。
- 历史选项卡通过图表展示提交历史，图表上方有一行工具栏区域，分为左侧和右侧。
    - 左侧为左对齐的工具组，分别是：分支筛选器。
        - 分支筛选器默认情况下是 “自动”，点击后会弹出下拉菜单，显示 “全部”、“自动” 两个特殊项，并有分割线分割，后跟所有分支项。
        - 每一项前面有复选框。
        - 当选择 “全部” 或者 “自动” 时，取消其它所有复选框的选择。
        - “全部”：即显示所有分支的提交历史。
        - “自动”：即显示当前所在分支的提交历史。
    - 右侧为右对齐的工具组，分别是：搜索框。
        - 搜索框默认情况下是空的，用户输入内容后会进行模糊搜索，搜索范围为提交信息、作者、提交内容。
- 点击任何提交记录会弹出从下方滑出的提交详情面板，半透明遮罩阻挡界面下层点击，点击遮罩部分关闭面板。
- 本地更改选项卡显示当前所有未提交的更改，分为两个区域：
    - 左半部分是更改文件列表，列表的形式展示所有未提交的更改，每个列表项前面有个复选框，列表上方有一行工具栏，左对齐是一个复选框，有三个状态（全选、全不选、部分选），右对齐有一个搜索框。
        - 列表项有两个状态组，一个是选中状态，一个是勾选状态。
        - 右键列表项会弹出菜单，菜单内容为：还原更改、储藏更改、勾选、取消勾选，针对的是所有选中的列表项。
        - 搜索框：默认情况下是空的，用户输入内容后会进行模糊搜索，搜索范围为文件名和文件内容。
        - 列表下方是按钮组，右对齐的按钮组分别是：提交按钮。
        - 文件列表的勾选状态要尽量在内部操作中保持，比如自动储藏并恢复本地更改时，恢复后要保持原来的勾选状态。
    - 右半部分是 Diff 区域，显示当前选中更改文件的 Diff 内容。

# 冲突解决界面

该界面会展示所有有冲突的文件列表，并且展示选中文件的冲突内容。
需要标明哪个是他人的更改，哪个是自己的更改，比如软件内部的自动储藏时，储藏的内容称为自己的更改，当前分支的内容称为他人的更改。
但是手动应用储藏时，则直接称为储藏的内容，当前分支的内容。
该界面需要能够多选、全选、反选 + 使用任何一边的更改 + 进行手动编辑 + 保存更改。

# 设置界面

弹出一个 Modal 窗口，左侧栏和右侧面板。

设置分为：

- 基本设置
    - Git 用户信息：用户名、邮箱地址。
    - SSH Key：显示当前使用的 SSH Key，并有复制按钮和生成按钮。
- 项目设置
    - 远程仓库地址：显示当前项目的远程仓库地址，并有复制按钮和修改按钮。
    - 自动跟踪设置：可以设置 A 分支追踪 B 分支，设置了自动追踪的 A 分支在执行同步操作时会使用不同的流程。
        - 这个设置是一个列表，有增删改按钮，点击后弹出一个 Modal 窗口让用户选择 A 分支和 B 分支。
