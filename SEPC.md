# 介绍

一个美术人员使用的 Git 桌面端工具，名称叫 Artistic Git。

- 它基于 Tauri + React + shadcn ui + lucide icon + Vite + Vitest，全部技术栈均使用最新稳定版本。
- 支持自动更新，运行时每小时检查更新，并有主动检查更新按钮。
- 支持 Github Actions 编译构建，并支持手动触发和推送提交自动触发。
- 版本号自动递增，自动上传二进制文件到 Github Releases，自动收集提交信息作为更新日志。
- 需要编译所有平台的产物。
- 对于核心逻辑需要编写严格的单元测试和端到端测试（不关心测试时长，用真实临时项目测试，而不是 fake/mock 命令），该工具涉及到资产丢失问题，需要首要保证安全性和稳定性。
- 界面响应式设计，支持任意窗口尺寸下最佳显示，默认尺寸 1280x720。

# 设计总则

- 设计上应简单人性化，美术人员对 Git 技术并不了解，在使用过程中应尽量避免存在对 Git 概念和操作理解的要求，将主要工作流设计为可视化操作，在工具内部自动执行 Git 命令。
- 界面应简洁精美，有苹果和 Linear 的设计感，多使用图标以增加亲近感和友好性。
- 合理使用符合直觉、苹果设计风格的动画效果。
- 记录详细错误日志，发生错误时弹出专门的错误详情弹窗，并支持复制错误信息和打开日志文件所在目录方便发送给开发人员。
- 超出长度的文本使用简略显示，如："/sdfsdf/sdfsd/fsdf..."， 鼠标悬停弹出 tooltip 显示完整路径。
- 保持界面元素自身文案的简练性，需要更多说明则使用悬停 tooltip 展示。
- 支持同时打开多个窗口管理多个项目。
- 支持中英文双语切换，自动根据系统语言切换，支持手动切换语言。
- 支持深色和浅色主题切换，自动根据系统主题切换，支持手动切换主题。

# 简化 Git 概念

- 工具内嵌 Git，所有 Git 操作均使用内嵌 Git 执行。
- 远程仓库地址：项目只能有一个远程仓库地址，名称为 `origin`，工具不考虑多远程仓库的情况。
- 分支：工具不存在本地分支和远程分支的概念，工具默认本地分支不可能存在未推送的提交，如果存在会直接报错，
禁止打开项目，退回到起始界面，要求用户先处理本地分支未推送的提交。
所以展示分支信息时直接展示其远程分支的信息，且内部会实时 Fetch 远程分支。

# 起始界面

- 打开该工具判断是否首次进入，是则打开向导界面，否则打开起始界面。
- 左侧是两个带图标的文字按钮，拉取项目、打开项目。
- 右侧是之前打开过的项目列表，每个列表项左边是图标，然后经跟着图标的是左对齐的两行文本：目录名称和灰色字体的详细路径。点击可以打开项目。
鼠标悬停在列表项上，会出现一排右对齐的按钮（仅图标）组，分别是：删除。
列表的最底部有清空历史记录的文字链接形式的按钮。
- 界面的右下角是右对齐的按钮（仅图标）组，从左到右是：向导、设置。

# 向导界面

- 作者信息
    - 要求用户输入 Git 使用的用户名和邮箱地址，这会存储起来，工具内部的操作均使用这些信息。
- SSH Key
    - 检测用户是否有 Git 能使用的 SSH Key，如果没有则可视化界面引导用户生成 SSH Key。
    - 生成后、或已有 SSH Key 的情况下，显示复制 SSH Key 的按钮，并附上将 Key 添加到 Git 平台的小字说明文本，点击后复制到剪贴板。

# 主界面

- 选择项目后进入主界面。
- 主界面分为左侧栏和主面板区域。
- 左侧栏内容从上到下排列：
    - 项目信息区域，从左到右是图标；文本区域，文本区域是左对齐的两行文本：目录名称和灰色字体的详细路径；右对齐的一个同步按钮（仅图标），如果存在可同步的内容则会变成橙色，否则是正常主色。
    - 分支信息区域，列表的形式展示分支，每个列表项占一行，从左到右是：选中标记（点）、图标、分支名称，然后是右对齐的待拉取提交数 Badge（如果没有待拉取提交则不显示）。
    鼠标悬停在列表项上，会出现一排右对齐的按钮（仅图标）组，分别是：同步。
    点击会选中该分支，首次打开项目时是未选中任何分支的状态，后续会记录上次选中的分支，下次打开项目时会自动选中该分支。
    - 储藏信息区域，列表的形式展示储藏记录，每个列表项占一行，从左到右是：图标、储藏名称，然后是右对齐的储藏时间。
    鼠标悬停在列表项上，会出现一排右对齐的按钮（仅图标）组，分别是：应用、删除、详情。
    - 分支信息和储藏信息区域均可以折叠，并有搜索框可以进行模糊搜索。
    未搜索到时会显示 “未搜索到相关内容” 的提示。
    - 审查模式按钮区域，有一个撑满该区域的青色渐变的 “进入审查模式” 按钮，点击后进入审查模式。
    - 更多操作按钮区域，从左到右排列的按钮（仅图标）组：设置。
    - 项目信息区域、审查模式按钮区域、更多操作按钮区域是固定高度的，分支信息和储藏信息区域本身的高度是自动撑满可用左侧栏高度的，并且有 handle 可以调节区域占用的尺寸，列表内容过多时会出现滚动条。
    调节的比例会被记录，下次打开项目时会恢复上次的比例。

# 工作流

- 首次使用工具：向导界面
- 首次拉取项目：起始界面
- 同步至远程仓库：在主界面点击 “同步” 按钮。
- 提交更改：在主界面勾选要提交的修改，输入提交信息，点击 “提交” 按钮。
- 还原更改：在主界面勾选要还原的修改，右键点击菜单中的 “还原” 按钮。
- 撤回提交：在主界面勾选要撤回的提交记录，右键点击菜单中的 “创建撤回提交” 按钮。
- 储藏更改：在主界面勾选要储藏的修改，输入储藏信息，点击 “储藏” 按钮。
- 恢复储藏：在主界面勾选要恢复的储藏记录，右键点击菜单中的 “恢复” 按钮。
- 删除储藏：在主界面勾选要删除的储藏记录，右键点击菜单中的 “删除” 按钮。
- 审查模式：在主界面点击 “审查模式” 按钮，进入审查模式，此时覆盖层阻止所有操作，中间有一个 “退出审查模式” 按钮。
